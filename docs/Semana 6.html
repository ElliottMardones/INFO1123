Herencia de Python

La herencia nos permite definir una clase que hereda todos los mtodos y propiedades de otra clase.

Clase padre es la clase que se hereda, tambin llamada clase base o principal.

Clase hijo es la clase que hereda de otra clase, tambin llamada clase derivada.

Clase padre

Cualquier clase puede ser una clase principal, por lo que la sintaxis es la misma que la de crear cualquier otra clase

In[]:
class Person:
  def __init__(self, fname, lname):
    self.firstname = fname
    self.lastname = lname

  def printname(self):
    print(self.firstname, self.lastname)


x = Person("John", "Doe")
x.printname()

John Doe

Clase hijo

Para crear una clase que herede la funcionalidad de otra clase, enve la clase principal como parmetro al crear la clase secundaria:

Ejemplo:

In[]:
class Student(Person):
  pass
  # Utilice la palabra 'pass' cuando no desee agregar ninguna otra propiedad o mtodo a la clase.

Ahora la clase Student tiene las mismas propiedades y mtodos que la clase Person.

Ejemplo

Utilice la clase Student para crear un objeto y luego ejecute el mtodo printname:

In[]:
x = Student("Mike", "Olsen")
x.printname()

Mike Olsen

Agregue la funcin init

Hasta ahora hemos creado una clase hija que hereda las propiedades y mtodos de su clase padre. Queremos agregar la funcin init() a la clase secundaria (en lugar de la palabra clave pass).

Ejemplo

In[]:
class Student(Person):
  def __init__(self, fname, lname):
    # agrega algunas propiedades
    self.fname = fname
    self.lname = lname

IMPORTANTE

Cuando agrega la funcin init(), la clase secundaria ya no heredar la funcin init() del padre.

La funcin init del hijo anula la herencia de la funcion init del padre.

Para mantener la herencia de la funcin init() del padre, agregue una llamada a la funcin init() del padre:

In[]:
class Student(Person):
  def __init__(self, fname, lname):
    Person.__init__(self, fname, lname)

x2 = Student('Jeffrey', 'Kaplan')
x2.printname()

Jeffrey Kaplan

Utilice la funcin super()

Python tambin tiene una funcin super() que har que la clase hija herede todos los mtodos y propiedades de su clase padre:

In[]:
class Student(Person):
  def __init__(self, fname, lname):
    super().__init__(fname, lname)

Al utilizar la funcin super(), no es necesario utilizar el nombre del elemento principal, este heredar automticamente los mtodos y propiedades de su elemento principal.

Si agrega un mtodo en la clase secundaria con el mismo nombre que una funcin en la clase principal, se anular la herencia del mtodo principal.

Ejemplo utilizando herencia en Python

Paso 1: Definicin de la clase Base

Vamos a crear una clase base llamada Persona que contendr dos atributos: nombre y edad.

In[]:
class Persona:
    def __init__(self, nombre, edad):
        self.nombre = nombre
        self.edad = edad

    def mostrar_informacion(self):
        print(f"Nombre: {self.nombre}")
        print(f"Edad: {self.edad}")

Paso 2: Creacin de una clase Derivada (hija)

vamos a crear una clase derivada llamada Empleado que heredar los atributos y mtodos de la clase Persona. Adems, aadiremos un nuevo atributo: salario.

In[]:
class Empleado(Persona):
    def __init__(self, nombre, edad, salario):
        super().__init__(nombre, edad)
        self.salario = salario

    def mostrar_informacion(self):
        super().mostrar_informacion()
        print(f"Salario: {self.salario}")

En este cdigo, hemos utilizado super() para llamar al mtodo __init__ de la clase base y hemos sobreescrito el mtodo mostrar_informacion para incluir informacin adicional sobre el salario.

Paso 3: Creacin de Instancias y Prueba de los Mtodos

Ahora crearemos instancias de las clases Persona y Empleado, y luego llamaremos al mtodo mostrar_informacion para cada una de ellas.

In[]:
# Creando una instancia de la clase Persona
persona1 = Persona("Alice", 30)
persona1.mostrar_informacion()

# Creando una instancia de la clase Empleado
empleado1 = Empleado("Bob", 40, 50000)
empleado1.mostrar_informacion()

Polimorfismo de Python

La palabra "polimorfismo" significa "muchas formas" y en programacin se refiere a mtodos/funciones/operadores con el mismo nombre que se pueden ejecutar en muchos objetos o clases.

Polimorfismo de funcin

Un ejemplo de una funcin de Python que se puede utilizar en diferentes objetos es la funcin len().

Ejemplos

  1. String
In[]:
x = "Hello World!"

print(len(x))

12
  1. Tuplas
In[]:
mytuple = ("apple", "banana", "cherry")

print(len(mytuple))

3
  1. Diccionarios
In[]:
thisdict = {
  "brand": "Ford",
  "model": "Mustang",
  "year": 1964
}

print(len(thisdict))

3

Polimorfismo de clase

El polimorfismo se usa a menudo en mtodos de clase, donde podemos tener varias clases con el mismo nombre de mtodo.

Por ejemplo, digamos que tenemos tres clases: Car, Boat y Plane, y todas tienen un mtodo llamado move():

    print("Drive!")

  def __init__(self, brand, model):
    self.brand = brand
    self.model = model

  def move(self):
    print("Sail!")

  def __init__(self, brand, model):
    self.brand = brand
    self.model = model

Crea una clase Vehiculo con atributos color y numero_de_ruedas. Luego, crea clases hijo como Coche, Moto y Bicicleta, que hereden de Vehiculo. Cada una de estas clases hijo debe tener atributos adicionales nicos.

In[]:
# Clase Padre
class Vehiculo:
    def __init__(self, color, numero_de_ruedas):
        self.color = color
        self.numero_de_ruedas = numero_de_ruedas

    def describir(self):
        print(f"Soy un vehculo de color {self.color} y tengo {self.numero_de_ruedas} ruedas.")

# Clase Hija: Coche
class Coche(Vehiculo):
    def __init__(self, color, numero_de_ruedas, velocidad_maxima):
        super().__init__(color, numero_de_ruedas)
        self.velocidad_maxima = velocidad_maxima

    def describir(self):
        print(f"Soy un coche de color {self.color} con una velocidad mxima de {self.velocidad_maxima} km/h.")

# Clase Hija: Moto
class Moto(Vehiculo):
    def __init__(self, color, numero_de_ruedas, tipo):
        super().__init__(color, numero_de_ruedas)
        self.tipo = tipo  # Deportiva, Turismo, etc.

    def describir(self):
        print(f"Soy una moto {self.tipo} de color {self.color}.")

# Clase Hija: Bicicleta
class Bicicleta(Vehiculo):
    def __init__(self, color, numero_de_ruedas, tipo):
        super().__init__(color, numero_de_ruedas)
        self.tipo = tipo  # Montaa, Carretera, etc.

    def describir(self):
        print(f"Soy una bicicleta {self.tipo} de color {self.color}.")
In[]:
# Creando instancias y describiendo cada una
coche = Coche("rojo", 4, 200)
coche.describir()

moto = Moto("azul", 2, "deportiva")
moto.describir()

bicicleta = Bicicleta("verde", 2, "de montaa")
bicicleta.describir()

Diagrama de clases

Un diagrama de clases en Lenguaje Unificado de Modelado (UML) es un tipo de digrama de estructura esttica que describe la estructura de un sistema mostrando las clases del sistema, sus atributos, mtodos y las relaciones entre los objetos.

Visibilidad

Para especificar la visibilidad de un miembro de la clase (es decir, cualquier atributo o mtodo), se coloca uno de los siguientes signos delante de ese miembro:

Relaciones

Una relacin es un trmino general que abarca los tipos especficos de conexiones lgicas que se pueden encontrar en los diagramas de clases y objetos. UML presenta las siguientes relaciones:

  • Asociacin representa a una familia de enlaces. Una asociacin binaria (entre dos clases) normalmente se representa con una lnea continua. Una misma asociacin puede relacionar cualquier nmero de clases. Una asociacin que relacione tres clases se llama asociacin ternaria. A una asociacin se le puede asignar un nombre, y en sus extremos se puede hacer indicaciones, como el rol que desempea la asociacin, los nombres de las clases relacionadas, su multiplicidad, su visibilidad, y otras propiedades. Hay cuatro tipos diferentes de asociacin: bidireccional, unidireccional, agregacin (en la que se incluye la composicin) y reflexiva. Las asociaciones unidireccional y bidireccional son las ms comunes. Por ejemplo, una clase vuelo se asocia con una clase avin de forma bidireccional. La asociacin representa la relacin esttica que comparten los objetos de ambas clases.

  • Agregacin o agrupacin es una variante de la relacin de asociacin tiene un: la agregacin es ms especfica que la asociacin. Se trata de una asociacin que representa una relacin de tipo parte-todo o parte-de.

Como se puede ver en la imagen del ejemplo (en ingls), un Profesor 'tiene una' clase a la que ensea.

Al ser un tipo de asociacin, una agregacin puede tener un nombre y las mismas indicaciones en los extremos de la lnea. Sin embargo, una agregacin no puede incluir ms de dos clases; debe ser una asociacin binaria.

Una agregacin se puede dar cuando una clase es una coleccin o un contenedor de otras clases, pero a su vez, el tiempo de vida de las clases contenidas no tienen una dependencia fuerte del tiempo de vida de la clase contenedora (de el todo). Es decir, el contenido de la clase contenedora no se destruye automticamente cuando desaparece dicha clase.

  • Composicin La representacin en UML de una relacin de composicin es mostrada con una figura de diamante relleno del lado de la clase contenedora, es decir al final de la lnea que conecta la clase contenido con la clase contenedor.

Diferencias entre Composicin y Agregacin

  • Relacin de Composicin
  1. Cuando intentamos representar un todo y sus partes. Ejemplo, un motor es una parte de un coche.
  2. Cuando se elimina el contenedor, el contenido tambin es eliminado. Ejemplo, si eliminamos una universidad eliminamos igualmente sus departamentos.

Relacin de Agregacin (o Agrupacin)

  1. Cuando representamos las relaciones en un software o base de datos. Ejemplo, el modelo de motor MTR01 es parte del coche MC01. Como tal, el motor MTR01 puede ser parte de cualquier otro modelo de coche, es decir si eliminamos el coche MC01 no es necesario eliminar el motor pues podemos usarlo en otro modelo.
  2. Cuando el contenedor es eliminado, el contenido usualmente no es destruido. Ejemplo, un profesor tiene estudiantes, cuando el profesor muere los estudiantes no mueren con l o ella.
  • Herencia Este tipo de relaciones permiten que una clase reciba los atributos y mtodos de otra clase (padre o super clase). Estos atributos y mtodos recibidos se suman a los que la clase tiene por s misma.
